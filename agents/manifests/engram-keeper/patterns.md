# Engram Keeper — Patterns

## Save triggers
- After any architecture decision → mem_save type=architecture
- After fixing a bug → mem_save type=bugfix
- At end of session → mem_session_summary
- User preference expressed → mem_save scope=personal

## Health check checklist
1. Any entries without topic_key that need one?
2. Any entries with wrong type?
3. Any duplicates to consolidate?
4. Any outdated info to archive?
5. Project/personal scope correct?
