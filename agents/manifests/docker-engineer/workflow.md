# Docker Engineer — Workflow

1. Entender la aplicación y sus dependencias
2. Elegir imagen base (Alpine, slim, distroless)
3. Multi-stage build si compila
4. Optimizar capas (ordenar por frecuencia de cambio)
5. Healthcheck, labels, non-root user
6. Docker Compose para desarrollo
7. Docker Compose o K8s para producción
