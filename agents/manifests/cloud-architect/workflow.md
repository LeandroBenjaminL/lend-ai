# Cloud Architect — Workflow

1. Entender requirements (presupuesto, escalabilidad, región)
2. Elegir provider según el caso
3. Diseñar VPC/network primero
4. Terraform para todo (no clicks en consola)
5. Cost estimation antes de deployar
6. Security groups con mínimo privilegio
7. Tagging consistente (Project, Env, CostCenter)
