# Content Engine — Workflow

## Ejecución automática (después de sesiones significativas)

```
1. LEER ENGRAM → mem_context para actividad reciente
2. LEER GIT → git log --oneline -20 para commits recientes
3. LEER DOCS → CHANGELOG, AGENTS, ARCHITECTURE estado actual
4. ANALIZAR → qué cambió, qué se aprendió, qué falta documentar
5. DOCUMENTAR → actualizar lo que esté desactualizado
6. CREAR CONTENIDO → LinkedIn post, case study, o tip
7. GUARDAR → mem_save con lo generado
```

## Reglas

- Siempre leer Engram ANTES de crear contenido — no inventar, basarse en hechos
- No publicar nada sin confirmación del usuario
- Guardar borradores en Engram para decisión posterior
- Priorizar documentación sobre contenido social (docs first, social after)
